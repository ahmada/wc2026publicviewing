## Rate Limiting KV Namespace
Create a Cloudflare KV namespace for IP-based rate limiting and wire it to the Pages project.

Steps:
1. Workers & Pages → KV → Create namespace (suggested name: tsa-rate-limit)
2. Pages project → Settings → Functions → KV namespace bindings
3. Add binding: variable name = RATE_LIMIT_KV, namespace = the one created above

Context: The /api/register endpoint already uses RATE_LIMIT_KV (3 requests/hr, 20/day per IP).
Rate limiting silently skips when the binding is absent, so this does not block launch —
it is a post-launch hardening step.

## WhatsApp Business Automation (WATI + WhatsApp Flows)
Build an automated WhatsApp registration flow that collects the same information as the web form,
using WATI as the BSP and WhatsApp Flows for the native in-chat form experience.

Steps:
1. Sign up at wati.io and migrate the existing WhatsApp Business number
2. Build the registration flow in WATI's flow builder (9 equivalent steps)
3. Point WATI's webhook at /api/register so submissions feed the same pipeline
4. Test end-to-end: WhatsApp submission → Google Sheet row + confirmation emails

Context: Existing /api/register endpoint and Google Sheets integration are already in place.
WATI webhook payload will need mapping to match the RegistrationSchema in src/types/registration.ts.
