# Lessons Learned

## Format: [date] | what went wrong | rule to prevent it

2026-05-19 | `npm create astro` won't scaffold into a non-empty directory | Scaffold to /tmp first, then copy files to target
2026-05-19 | `cookie` CJS package fails in Cloudflare Workerd's `runInlinedModule` | Create ESM shim at `src/shims/cookie.js` + alias it via `vite.resolve.alias`
2026-05-19 | `@tailwindcss/vite` triggers Rolldown's missing `tsconfigPaths` field error | Set `vite.resolve.tsconfigPaths: false` explicitly
2026-05-19 | Build fails: "rollupOptions.input should not be an html file when building for SSR" | Root cause: `@cloudflare/vite-plugin` imports top-level Vite 8 (`isRolldown=true`) and puts worker entry in `rolldownOptions.input`. Astro's Vite 7 reads `rollupOptions.input` → undefined → falls back to index.html. Fix: add `enforce: 'post'` Vite plugin that copies `rolldownOptions.input` → `rollupOptions.input` for each environment
2026-05-20 | Vite `configureServer` middleware never intercepts user requests in `@astrojs/cloudflare` v13 dev mode | `@cloudflare/vite-plugin` starts workerd (via miniflare) as the ACTUAL HTTP server on port 4321. Vite's Connect middleware is bypassed entirely for all user HTTP requests — only used for HMR/WebSocket on a different internal port. Never attempt to use `configureServer` to mock or intercept API calls in Cloudflare Astro dev mode.
2026-05-20 | `picomatch` CJS module fails in workerd `runInlinedModule` with "require_picomatch is not a function" | workerd's module runner uses `eval()` — Vite's `__commonJSMin` CJS wrapper pattern fails in this context. Fix: `vite.ssr.optimizeDeps.exclude: ['picomatch']` forces workerd to load it via native `import()` (requires `nodejs_compat` in wrangler.toml). Pass `configPath: 'wrangler.toml'` to the cloudflare adapter so dev workerd picks up the flag.
2026-05-20 | `Astro.locals.runtime.env has been removed in Astro v6` crashes API routes | `@astrojs/cloudflare` v13 removed `App.Locals.runtime.env` — the getter now throws. Use `import { env as cfEnv } from 'cloudflare:workers'` instead, which works in both dev (miniflare) and production. Remove `App.Locals.runtime` from `env.d.ts`.
2026-05-20 | `res.json()` called before `res.ok` check causes "Network error" when server returns non-JSON (e.g. 501 HTML) | Always wrap `await res.json()` in try/catch with a default value, then check `res.ok`. Pattern: `let json = { ok: false }; try { json = await res.json(); } catch {} if (!res.ok) { ... }`
